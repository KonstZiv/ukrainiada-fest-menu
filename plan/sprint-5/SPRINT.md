## Детально для виконавця

### Флоу передачі страви

```
Кухар: натискає "Передати офіціанту X" → генерується QR (містить transfer token)
Офіціант: сканує QR → бачить "Підтвердити прийом Борщ від Іванченко" → натискає
    ↓
KitchenTicket.status = DONE, transfer підтверджений

Fallback (без QR):
Кухар: натискає "Передав офіціанту" → одностороннє підтвердження (без scan)
```

### Transfer token

Одноразовий UUID токен, зберігається в `KitchenHandoff` моделі.
Протермінований через `settings.HANDOFF_TOKEN_TTL` секунд (default: 120).

- [ ] Fallback: кухар сам відмічає "передав" без сканування
