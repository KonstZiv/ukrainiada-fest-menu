## 0. Prerequisites

### Users (створити через Django admin `/admin/`)

| Email | Role | Пароль | Призначення |
|-------|------|--------|-------------|
| `cook1@test.com` | kitchen | test123 | Кухар 1 — привʼязаний до 2-3 страв |
| `cook2@test.com` | kitchen | test123 | Кухар 2 — привʼязаний до інших страв |
| `chef@test.com` | kitchen_supervisor | test123 | Шеф-повар |
| `waiter@test.com` | waiter | test123 | Офіціант |
| `manager@test.com` | manager | test123 | Менеджер |
| `visitor@test.com` | visitor | test123 | Відвідувач |

### KitchenAssignment (через `/admin/kitchen/kitchenassignment/`)

- cook1 → Борщ, Вареники, Салат
- cook2 → Шашлик, Деруни
- chef → всі страви (або кілька для перевірки)

### Інфраструктура

```bash
docker compose -f compose.dev.yml --env-file .env-dev up -d   # PostgreSQL + Redis
uv run python manage.py migrate
uv run python manage.py loaddata fixtures/menu_data.json       # якщо потрібно
uv run python manage.py runserver
```

### Інструменти

- Chrome DevTools → responsive mode (iPhone SE 375px, iPad 768px, Desktop 1280px)
- 2-3 вкладки браузера (різні ролі одночасно)
- Секундомір або `KITCHEN_WARN_MINUTES=1` + `KITCHEN_TIMEOUT=2` у `.env-dev` для швидшого тестування urgency
