- [ ] **Офіціант:** перейти на деталі замовлення (order_scan)
- [ ] **Офіціант:** секція "Позачергові дії" — натиснути "Доставлено (без кухні)"

- [ ] **Офіціант:** прийняти оплату
- [ ] **Менеджер:** dashboard — колонка "помилки": скарги (badge червоний) + помилки кроків (badge жовтий) у Дмитра

## Фаза 5: StepEscalation — blame tracking (timeout ескалації)

> **Підготовка:** Celery Beat має бути запущений (`uv run celery -A core_settings beat -l info`
> + `uv run celery -A core_settings worker -l info`). Або можна тестувати через Django Admin,
> вручну створюючи StepEscalation записи.

### 5.1 Timeout: SUBMITTED → ACCEPTED (ACCEPT_TIMEOUT = 5 хв)
- [ ] **Гість:** створити замовлення
- [ ] **Офіціант:** НЕ брати замовлення
- [ ] Почекати >5 хв (або в Django shell: `from orders.tasks import escalate_unaccepted_orders; escalate_unaccepted_orders()`)
- [ ] **Admin** `/admin/orders/stepescalation/` — з'явився запис: step=submit_accept, level=1, owner_role=senior_waiter
- [ ] **Офіціант:** взяти замовлення → запис у Admin отримує resolved_at

### 5.2 Timeout: ACCEPTED → VERIFIED (VERIFY_TIMEOUT = 5 хв)
- [ ] **Офіціант:** взяти замовлення, НЕ верифікувати
- [ ] Почекати >5 хв (або shell: `from orders.tasks import escalate_unverified_orders; escalate_unverified_orders()`)
- [ ] **Admin:** step=accept_verify, level=1, owner=Дмитро
- [ ] **Офіціант:** верифікувати → resolved_at заповнено

### 5.3 Timeout: TAKEN → DONE (COOKING_TIMEOUT = 15 хв)
- [ ] **Кухня:** взяти тікет, НЕ завершувати
- [ ] Shell: `from kitchen.tasks import escalate_cooking_tickets; escalate_cooking_tickets()`
- [ ] **Admin:** step=taken_done, level=1, owner=Валентина

### 5.4 Timeout: DONE → handoff (HANDOFF_TIMEOUT = 10 хв)
- [ ] **Кухня:** завершити тікет, НЕ передавати вручну
- [ ] Shell: `from kitchen.tasks import escalate_handoff_tickets; escalate_handoff_tickets()`
- [ ] **Admin:** step=done_handoff, level=1, owner=Валентина

### 5.5 Promotion: senior → manager (SENIOR_RESPONSE_TIMEOUT = 10 хв)
- [ ] Мати нерозв'язану level=1 ескалацію старшу за 10 хв
- [ ] Запустити відповідний task повторно
- [ ] **Admin:** з'явився запис level=2, caused_by вказує на власника рівня 1

### 5.6 Dashboard stats — "помилки"
- [ ] **Менеджер:** dashboard → акордеон Офіціанти: колонка "помилки" показує кількість StepEscalation (accept_verify + deliver_pay) у жовтому badge
- [ ] **Менеджер:** dashboard → акордеон Кухня: колонка "помилки" показує кількість StepEscalation (taken_done + done_handoff) у червоному badge
- [ ] При 0 помилок — зелена галочка ✓

### 5.7 Admin — StepEscalation
- [ ] `/admin/orders/stepescalation/` — список відображається
- [ ] Фільтри step / level працюють
- [ ] Кожен запис має order або ticket посилання
